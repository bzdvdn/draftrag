# Retrieval: дедупликация источников (v1)

## Scope Snapshot

- In scope: добавить опциональную дедупликацию retrieval результата, чтобы источники не повторялись (по `ParentID` и/или контенту), сохраняя приоритет более релевантных чанков.
- Out of scope: семантическая дедупликация через вторую модель, кластеризация, сложные heuristics “near-duplicate”.

## Цель

Повысить полезность “Sources” и качество prompt: меньше повторов, больше разнообразия контекста.

## Основной сценарий

1. `VectorStore.Search` возвращает топ-K чанков, где несколько могут принадлежать одному документу или быть почти одинаковыми.
2. Pipeline применяет дедупликацию (если включено) перед построением prompt.
3. Пользователь получает более разнообразные источники.

## Контекст

- В проекте уже есть `RetrievalResult` (чанки + score + query text) и pipeline-методы, которые используют retrieval результат для построения prompt.
- В реальных данных topK часто содержит несколько чанков из одного и того же документа, что снижает разнообразие контекста и полезность UI “Sources”.
- Дедупликация должна быть опциональной и не ломать существующее поведение по умолчанию.

## Требования

- RQ-001 ДОЛЖЕН существовать механизм включения дедупликации (через config/options pipeline).
- RQ-002 Дедупликация ДОЛЖНА поддерживать режим “по ParentID”: оставить лучший (max score) чанк на документ.
- RQ-003 Дедупликация ДОЛЖНА сохранять порядок по релевантности (стабильная сортировка: score desc, tie-breaker стабильный).
- RQ-004 Дедупликация НЕ ДОЛЖНА менять поведение по умолчанию, если выключена.
- RQ-005 Тесты ДОЛЖНЫ проходить без внешней сети.

## Критерии приемки

### AC-001 Дедупликация по ParentID работает

- **Given** retrieval result содержит несколько чанков с одинаковым `ParentID`
- **When** включена дедупликация по `ParentID`
- **Then** в итоговом result остаётся не более одного чанка на `ParentID` (лучший по score)

### AC-002 По умолчанию ничего не меняется

- **Given** дедупликация выключена
- **When** выполняется Answer/Query
- **Then** результат идентичен текущему поведению
