# Qdrant vector store

## Scope Snapshot

- **In scope**: Реализация бэкенда Qdrant для интерфейсов `VectorStore` и `VectorStoreWithFilters` с поддержкой payload-фильтров.
- **Out of scope**: Гибридный поиск (BM25), поддержка других векторных БД, кластеризация Qdrant, тюнинг производительности.

## Цель

Пользователи draftRAG получают production-ready интеграцию с Qdrant — популярной векторной БД с открытым исходным кодом. Это позволяет использовать Qdrant как бэкенд для хранения и поиска эмбеддингов в RAG-приложениях с поддержкой фильтрации по метаданным.

## Основной сценарий

1. Пользователь создаёт клиент Qdrant через factory-функцию, передавая URL и опциональные настройки.
2. Пользователь вызывает миграцию для создания коллекции с нужной размерностью векторов.
3. Пользователь использует клиент как `VectorStore` для upsert/delete/search операций.
4. При необходимости фильтрации по метаданным — клиент используется как `VectorStoreWithFilters`.
5. Поиск с фильтром возвращает только чанки, удовлетворяющие payload-условиям Qdrant.

## Scope

- REST API клиент для Qdrant (без gRPC)
- Реализация интерфейса `VectorStore`
- Реализация интерфейса `VectorStoreWithFilters`
- Поддержка payload-фильтров Qdrant (match, range, keyword)
- Factory-функции для создания клиента
- Миграции: создание/удаление коллекции
- Валидация размерности векторов
- Обработка ошибок Qdrant API

## Контекст

- Qdrant предоставляет REST API для всех операций; gRPC опционален и вне scope
- Payload-фильтры Qdrant мощнее базового `ParentIDFilter`, требуется маппинг
- Размерность векторов фиксируется при создании коллекции
- Доменные модели (`Chunk`, `RetrievalResult`) уже определены в `internal/domain`
- Интерфейсы `VectorStore` и `VectorStoreWithFilters` зафиксированы в конституции

## Требования

- **RQ-001** Клиент ДОЛЖЕН реализовывать интерфейс `VectorStore` из `internal/domain`
- **RQ-002** Клиент ДОЛЖЕН реализовывать интерфейс `VectorStoreWithFilters` из `internal/domain`
- **RQ-003** Factory-функция `NewQdrantStore` ДОЛЖНА принимать URL, коллекцию и размерность векторов
- **RQ-004** Метод `CreateCollection` ДОЛЖЕН создавать коллекцию в Qdrant с указанной размерностью
- **RQ-005** Метод `DeleteCollection` ДОЛЖЕН удалять коллекцию из Qdrant
- **RQ-006** Метод `Upsert` ДОЛЖЕН сохранять чанк с эмбеддингом и payload (ID, Content, ParentID, Metadata)
- **RQ-007** Метод `Search` ДОЛЖЕН выполнять векторный поиск и возвращать `RetrievalResult`
- **RQ-008** Метод `SearchWithFilter` ДОЛЖЕН фильтровать по ParentID через payload-фильтр Qdrant
- **RQ-009** Метод `SearchWithMetadataFilter` ДОЛЖЕН фильтровать по произвольным метаданным через payload
- **RQ-010** Клиент ДОЛЖЕН маппить `MetadataFilter` на payload-фильтры Qdrant (match условия)
- **RQ-011** Все методы ДОЛЖНЫ принимать `context.Context` для таймаутов и отмены

## Вне scope

- gRPC клиент для Qdrant
- Гибридный поиск (BM25 + семантический) — требует отдельного интерфейса `HybridSearcher`
- Аутентификация API ключами (базовая поддержка через HTTP headers в будущем)
- Управление кластером Qdrant (шардирование, репликация)
- Снапшоты и бэкапы коллекций
- Квантизация векторов
- Поддержка sparse vectors

## Критерии приемки

### AC-001 Базовый векторный поиск

- **Почему важно**: Core-функциональность VectorStore — поиск похожих чанков
- **Given** созданная коллекция с индексированными чанками
- **When** вызывается `Search` с эмбеддингом запроса
- **Then** возвращаются релевантные чанки, отсортированные по score
- **Evidence**: тест `TestQdrantStore_Search` проходит, возвращает ожидаемые `RetrievedChunk`

### AC-002 Фильтрация по ParentID

- **Почему важно**: Позволяет ограничить поиск в пределах одного документа
- **Given** коллекция с чанками от разных документов
- **When** вызывается `SearchWithFilter` с `ParentIDFilter`
- **Then** возвращаются только чанки с matching ParentID
- **Evidence**: тест `TestQdrantStore_SearchWithParentIDFilter` проходит

### AC-003 Фильтрация по метаданным

- **Почему важно**: Позволяет фильтровать по произвольным полям (автор, тег, дата)
- **Given** коллекция с чанками, имеющими разные метаданные
- **When** вызывается `SearchWithMetadataFilter` с `MetadataFilter`
- **Then** возвращаются только чанки, где все пары key=value из фильтра совпадают
- **Evidence**: тест `TestQdrantStore_SearchWithMetadataFilter` проходит

### AC-004 Upsert и Delete операции

- **Почему важно**: CRUD-операции необходимы для управления индексом
- **Given** валидный `Chunk` с эмбеддингом
- **When** вызывается `Upsert`, затем `Delete` с тем же ID
- **Then** чанк сначала доступен в поиске, затем удалён
- **Evidence**: тест `TestQdrantStore_UpsertDelete` проходит, после Delete чанк не находится

### AC-005 Создание и удаление коллекции

- **Почему важно**: Миграции позволяют управлять схемой хранилища
- **Given** Qdrant сервер доступен
- **When** вызывается `CreateCollection` с размерностью 768
- **Then** коллекция создаётся с корректной конфигурацией distance=Cosine
- **Evidence**: коллекция видна в Qdrant dashboard, тест `TestQdrantStore_Migrations` проходит

### AC-006 Обработка ошибок API

- **Почему важно**: Пользователь должен понимать причину сбоев
- **Given** неверный URL или несуществующая коллекция
- **When** вызывается любой метод
- **Then** возвращается информативная ошибка с деталями от Qdrant
- **Evidence**: тесты на ошибки проходят, ошибки содержат HTTP status и message от API

## Допущения

- Qdrant сервер доступен по HTTP REST API на стандартном порту (6333)
- Размерность векторов известна заранее и фиксирована для коллекции
- Payload ограничен примитивными типами (string, number, bool) — без nested objects
- Qdrant версии 1.7+ (поддержка всех используемых endpoint)
- Таймауты операций контролируются через `context.Context` вызывающей стороны
- Для тестов используется либо мок HTTP сервер, либо запущенный Qdrant (Docker)

## Критерии успеха

- **SC-001** Поиск по 10k векторов выполняется за <100ms (95-й перцентиль)
- **SC-002** Покрытие тестами infrastructure-слоя ≥60%

## Краевые случаи

- Пустая коллекция: `Search` возвращает пустой `RetrievalResult` без ошибок
- Несовпадающая размерность вектора: ошибка `ErrEmbeddingDimensionMismatch`
- Несуществующая коллекция: ошибка с HTTP 404 и понятным сообщением
- Пустой `MetadataFilter`: эквивалентно поиску без фильтра
- Timeout context: операция прерывается, возвращается `context.DeadlineExceeded`
