# Answer: inline citations в тексте ответа (v1)

## Scope Snapshot

- In scope: добавить опциональный режим, при котором answer возвращается с inline-цитатами вида `[1]`, `[2]`, а также маппинг этих номеров на retrieval sources.
- Out of scope: идеально точное связывание “факт → источник”, автоматическая подсветка span’ов, structured outputs, стриминг.

## Контекст

- В проекте уже есть `AnswerWithCitations`: метод возвращает `answer` + `RetrievalResult` (чанки + score), но сам текст ответа не содержит ссылок на источники.
- Для UX (например, в UI/чатах) нужен режим, где LLM добавляет ссылки на источники прямо в тексте ответа (`[n]`), а backend возвращает таблицу соответствия `n → chunk`.

## Цель

Сделать ответы “доказуемыми”: пользователь видит, какой источник поддерживает утверждения, без ручного сопоставления.

## Основной сценарий

1. Пользователь вызывает новый метод/режим Answer, включающий inline citations.
2. Pipeline формирует prompt, в котором просит LLM добавлять ссылки на источники `[n]`.
3. Пользователь получает:
   - `answer` (с `[n]`),
   - `sources` (как раньше),
   - `citations` (таблица соответствия `n → chunk`).

## Требования

- RQ-001 ДОЛЖЕН существовать публичный метод/опция v1 для “answer with inline citations”.
- RQ-002 Pipeline ДОЛЖЕН ограничивать максимальное число источников, доступных для цитирования (например topK либо отдельный лимит).
- RQ-003 ДОЛЖЕН возвращаться маппинг “citation number → source chunk” (чтобы UI мог показать источники).
- RQ-004 По умолчанию поведение Answer* НЕ ДОЛЖНО меняться.
- RQ-005 Тесты ДОЛЖНЫ проходить без внешней сети (использовать fake LLM).

## Критерии приемки

### AC-001 Возвращается маппинг citations → sources

- **Given** retrieval вернул N источников
- **When** вызывается “answer with inline citations”
- **Then** возвращается структура, позволяющая однозначно сопоставить `[1..M]` с конкретными чанками

### AC-002 Backward compatibility

- **Given** существующие Answer/AnswerWithCitations
- **When** проект собирается и запускаются тесты
- **Then** всё работает как раньше, новый режим добавлен аддитивно

## Открытые вопросы

- Должны ли мы “валидировать”, что LLM использовал только допустимые номера `[n]`, и что делать при нарушении? (v1: мягкая валидация и warning/error опционально)
