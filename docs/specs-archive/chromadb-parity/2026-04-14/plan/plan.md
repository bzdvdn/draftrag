# ChromaDB Parity — План реализации

## Phase Contract

- **Inputs:** spec из `docs/specs/chromadb-parity/spec.md`
- **Outputs:** план реализации с задачами
- **Stop if:** — (спецификация конкретна и реализуема)

---

## Цель

Достичь parity ChromaDB с другими vector store путём:
1. **Переименования API** для консистентности (breaking change)
2. **Документирования ограничений** — явно указать отсутствие hybrid search

Изменения затрагивают только публичный API (`pkg/draftrag`) и документацию. Внутренняя реализация (`internal/...`) не меняется.

---

## Scope

### Входит
- `pkg/draftrag/chromadb.go` — переименование функций
- `pkg/draftrag/chromadb_test.go` — обновление тестов
- `docs/compatibility.md` — документирование ограничений и обновление матрицы
- `ROADMAP.md` — корректировка статуса ChromaDB

### Не входит
- Изменения в `internal/infrastructure/vectorstore/chromadb.go`
- Реализация hybrid search для ChromaDB (невозможно нативно)
- Добавление новых функций (только переименование существующих)

## Acceptance Approach

### AC-001: API консистентность
- **Подход:** Простое переименование функций через IDE/редактор
- **Surfaces:** `pkg/draftrag/chromadb.go`
- **Проверка:** `go build ./...` проходит, grep не находит старых имён

### AC-002: Документация
- **Подход:** Редактирование markdown-файлов
- **Surfaces:** `docs/compatibility.md`
- **Проверка:** Визуальная проверка матрицы, `ChromaDB` в таблице управления коллекциями = ✓

### AC-003: Тестирование
- **Подход:** Глобальная замена в тестовом файле
- **Surfaces:** `pkg/draftrag/chromadb_test.go`
- **Проверка:** `go test ./...` проходит

### AC-004: ROADMAP
- **Подход:** Редактирование пункта о ChromaDB
- **Surfaces:** `ROADMAP.md`
- **Проверка:** Пункт отражает реальность (миграции ✅, hybrid search — не планируется)

## Стратегия реализации

### DEC-001: Последовательное переименование
- **Why:** Go не поддерживает deprecated aliases без кода. Простое переименование — чистое решение.
- **Tradeoff:** Breaking change для любого существующего кода (не критично до релиза).
- **Affects:** `pkg/draftrag/chromadb.go`, `pkg/draftrag/chromadb_test.go`
- **Validation:** `go test ./...` зелёный

### DEC-002: Явное документирование ограничений
- **Why:** Пользователи должны знать, что ChromaDB не подходит для hybrid search.
- **Tradeoff:** Может оттолкнуть пользователей от ChromaDB, но это честность.
- **Affects:** `docs/compatibility.md`
- **Validation:** Матрица показывает `—` для hybrid search у ChromaDB

### MVP (Полный scope)

Все задачи можно выполнить за один коммит — изменения локальны и независимы:

1. Переименование функций в `chromadb.go`

3. Обновление `compatibility.md`

**Критерий готовности:**
- `go test ./...` проходит
- `go build ./...` проходит
