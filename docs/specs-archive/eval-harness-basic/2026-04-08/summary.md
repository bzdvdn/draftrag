- snapshot: добавлен базовый eval harness для retrieval (hit@k, MRR) с детерминированными тестами
- slug: eval-harness-basic

## Причина

Нужно измерять качество retrieval и ловить регрессии при изменениях (дедуп/ранжирование/контекст). В v1 достаточно базовых метрик без LLM.

## Результат

- Новый публичный пакет `pkg/draftrag/eval` с `Run`, моделями данных и метриками.
- Unit-тест на синтетическом датасете фиксирует ожидаемые hit@k и MRR.

## Продолжение

- Возможное расширение: дополнительные метрики (например nDCG), оценка answer качества и импорт/экспорт датасетов (не в v1).
