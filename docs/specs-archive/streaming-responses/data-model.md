# Streaming ответов — Data Model

## Сущности

Нет новых persisted сущностей. Streaming — runtime-only концепция.

## Runtime-only структуры

### StreamingToken

Канал `chan string` передаёт текстовые чанки (tokens) от производителя к потребителю.

```go
// Псевдокод для документации — фактическая реализация через канал
type StreamingToken struct {
    // Text — очередной фрагмент ответа
    Text string
}
```

### StreamingResult

Результат streaming-операции — канал для чтения и возможная ошибка инициализации.

```go
// Паттерн возврата из GenerateStream
type StreamingResult interface {
    // Tokens — канал для чтения текстовых чанков
    // Закрывается при завершении генерации или ошибке
    Tokens() <-chan string
    
    // Err — ошибка инициализации streaming (не ошибка во время генерации)
    Err() error
}
```

**Примечание**: В Go идиоматичнее возвращать `(<-chan string, error)` напрямую, чем обёртку.

## Инварианты

- Канал закрывается при завершении генерации
- Канал закрывается при отмене контекста
- При ошибке инициации HTTP-запроса возвращается `error`, канал не создаётся
- При ошибке во время streaming'а (разрыв соединения) канал закрывается, ошибка логируется (потребитель видит только закрытие канала)

## Жизненный цикл

```
[Создание канала] → [Запуск горутины с SSE-чтением] → [Потребление токенов] → [Закрытие канала]
                      ↓
              [Отмена контекста] → [Закрытие канала, завершение горутины]
```

## Границы

- **Domain**: Нет state, только интерфейс `StreamingLLMProvider`
- **Application**: Runtime-only каналы, нет persistence
- **Infrastructure**: HTTP streaming response, буферизация минимальна (построчное чтение SSE)

## Отсутствующие сущности (осознанно)

- Нет persisted queue для streaming
- Нет кэширования streaming-ответов
- Нет replay-функциональности для streaming
