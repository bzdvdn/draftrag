# Ollama LLM + Embedder

## Scope Snapshot

- **In scope**: Поддержка локальных моделей через Ollama API для LLM (`/api/chat`) и Embedder (`/api/embeddings`)
- **Out of scope**: Управление lifecycle моделей Ollama (pull/run), stream processing для embedder, multi-modal модели

## Цель

Разработчики получают возможность использовать локальные LLM и embedding-модели через Ollama для оффлайн и приватных сценариев, где данные не должны покидать локальное окружение. Успех виден по работающей генерации текста и эмбеддингов через Ollama API с теми же интерфейсами, что и для облачных провайдеров.

## Основной сценарий

1. **Стартовая точка**: Пользователь запускает Ollama локально (`ollama run llama3.2`, `ollama pull nomic-embed-text`)
2. **Инициализация**: Пользователь создаёт `OllamaLLM` и/или `OllamaEmbedder` с указанием base URL (по умолчанию `http://localhost:11434`) и имени модели
3. **Основное действие**: Код вызывает `Generate()` для LLM или `Embed()` для embedder — запросы уходят на локальный Ollama API
4. **Результат**: Получен ответ от локальной модели без отправки данных во внешние сервисы
5. **Ошибка**: Если Ollama не запущен или модель не загружена — возвращается понятная ошибка с контекстом

## Scope

- Реализация `LLMProvider` интерфейса для Ollama через `/api/chat`
- Реализация `Embedder` интерфейса для Ollama через `/api/embeddings`
- Поддержка стандартных параметров (temperature, max tokens для LLM)
- Обработка ошибок Ollama API с понятными сообщениями
- Контекстная безопасность (context.Context propagation)
- Unit-тесты с мок-сервером
- Интеграционные тесты (опциональные, при наличии Ollama)

## Контекст

- Ollama API по структуре похож на OpenAI-compatible, но с другими endpoint'ами и форматом запросов/ответов
- `/api/chat` возвращает JSON с полем `message.content` вместо OpenAI-формата `choices[0].message.content`
- `/api/embeddings` принимает `prompt` вместо `input` и возвращает `embedding` напрямую вместо `data[0].embedding`
- Ollama не требует API key для локального использования
- Default base URL: `http://localhost:11434`
- Существующие интерфейсы `LLMProvider` и `Embedder` должны остаться неизменными

## Требования

- RQ-001 `OllamaLLM` должен реализовывать `domain.LLMProvider` интерфейс
- RQ-002 `OllamaLLM.Generate()` должен отправлять POST-запрос на `{baseURL}/api/chat` с форматом запроса Ollama
- RQ-003 `OllamaLLM` должен поддерживать параметры `model`, `temperature`, `max_tokens`
- RQ-004 `OllamaEmbedder` должен реализовывать `domain.Embedder` интерфейс
- RQ-005 `OllamaEmbedder.Embed()` должен отправлять POST-запрос на `{baseURL}/api/embeddings` с форматом запроса Ollama
- RQ-006 Обе реализации должны корректно обрабатывать HTTP-ошибки (4xx, 5xx) с маскированием чувствительных данных
- RQ-007 Обе реализации должны валидировать входные параметры (nil context, пустые строки)

## Вне scope

- Streaming-генерация для OllamaLLM (может быть добавлена позже через `StreamingLLMProvider`)
- Поддержка multi-modal моделей (image input)
- Автоматический pull моделей при отсутствии
- Управление Ollama сервером (запуск/остановка)
- Поддержка advanced параметров Ollama (format, options, keep_alive)
- Конвертация между форматами моделей

## Критерии приемки

### AC-001 LLM-генерация через Ollama

- **Почему это важно**: Пользователи могут генерировать ответы через локальные модели без отправки данных в облако
- **Given**: Ollama запущен локально с моделью `llama3.2`
- **When**: Пользователь вызывает `ollamaLLM.Generate(ctx, "system prompt", "user message")`
- **Then**: Запрос отправляется на `POST /api/chat` с корректным JSON-форматом Ollama
- **And**: Метод возвращает сгенерированный текст из поля `message.content` ответа
- **Evidence**: Успешный unit-test с мок-сервером, проверяющий формат запроса и парсинг ответа

### AC-002 Эмбеддинги через Ollama

- **Почему это важно**: RAG-система может использовать локальные embedding-модели без внешних API
- **Given**: Ollama запущен локально с моделью `nomic-embed-text`
- **When**: Пользователь вызывает `ollamaEmbedder.Embed(ctx, "текст для эмбеддинга")`
- **Then**: Запрос отправляется на `POST /api/embeddings` с полем `prompt` вместо `input`
- **And**: Метод возвращает валидный embedding-вектор из поля `embedding` ответа
- **Evidence**: Успешный unit-test с мок-сервером, проверяющий формат запроса и парсинг ответа

### AC-003 Обработка ошибок Ollama

- **Почему это важно**: Пользователи получают понятные сообщения при проблемах с локальным Ollama
- **Given**: Ollama не запущен или модель не найдена
- **When**: Пользователь вызывает метод `Generate()` или `Embed()`
- **Then**: Возвращается ошибка с контекстом (connection refused, model not found)
- **Evidence**: Тесты с мок-сервером возвращающим 404/500, проверка содержимого ошибки

### AC-004 Контекстная безопасность

- **Почему это важно**: Операции корректно прерываются при отмене контекста
- **Given**: Создан контекст с таймаутом или cancel-функцией
- **When**: Контекст отменяется во время выполнения HTTP-запроса
- **Then**: Операция прерывается и возвращает ошибку `context.Canceled` или `context.DeadlineExceeded`
- **Evidence**: Unit-test с таймаутом, проверяющий корректное поведение при отмене

### AC-005 Валидация входных данных

- **Почему это важно**: Раннее обнаружение ошибок предотвращает бесполезные HTTP-запросы
- **Given**: Пустой user message (только пробелы) или nil context
- **When**: Пользователь вызывает `Generate()` или `Embed()`
- **Then**: Метод возвращает ошибку валидации до отправки HTTP-запроса
- **Evidence**: Unit-тесты проверяют возврат ошибки для пустых строк и nil context

## Допущения

- Пользователь сам управляет установкой и запуском Ollama (вне scope библиотеки)
- Модели уже pulled в Ollama перед использованием (ollama pull <model>)
- Default base URL: `http://localhost:11434` (стандартный порт Ollama)
- API Ollama стабилен и соответствует документации на момент реализации
- Локальные модели достаточно быстры для синхронных вызовов без streaming
- Пользователи понимают ограничения локальных моделей (качество, скорость, ресурсы)

## Краевые случаи

- **Ollama не запущен**: Connection refused ошибка с понятным сообщением
- **Модель не найдена**: Ollama возвращает 404 — преобразуем в `ErrModelNotFound`
- **Пустой ответ от модели**: Обработка пустого `message.content` как ошибки
- **Невалидный embedding**: Проверка на NaN/Inf значения в векторе (как в OpenAI-совместимой реализации)
- **Длинный текст для эмбеддинга**: Ollama обрезает или возвращает ошибку — проксируем как есть
