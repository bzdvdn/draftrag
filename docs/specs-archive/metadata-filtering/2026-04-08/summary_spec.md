## Goal

Произвольная фильтрация по метаданным документа при семантическом поиске через расширение `VectorStoreWithFilters` и публичного API.

## Acceptance Criteria

| ID | Summary | Proof Signal |
|---|---|---|
| AC-001 | pgvector возвращает только совпадающие чанки | ID возвращённых чанков совпадают с ожидаемым множеством |
| AC-002 | Пустой фильтр не меняет результат поиска | Результаты идентичны вызову без фильтра |
| AC-003 | Публичный API передаёт фильтр сквозь pipeline | Мок или integration-тест подтверждает доставку фильтра |
| AC-004 | Нет совпадений — пустой результат без ошибки | `len(chunks)==0 && err==nil` |
| AC-005 | In-memory store компилируется и фильтрует | `go build ./...` и `go test ./...` exit 0 |

## Out of Scope

- Операторы сравнения (`>=`, `<=`, `LIKE`) — только точное совпадение
- Комбинация `ParentIDFilter` + `MetadataFilter` в одном вызове
- Реализации Qdrant, ChromaDB и других бэкендов
- Streaming-версии методов с фильтром
- Изменения схемы таблиц pgvector
