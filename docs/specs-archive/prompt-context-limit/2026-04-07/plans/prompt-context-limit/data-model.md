# Ограничение контекста в Prompt для draftRAG — Модель данных

## Scope

- Связанные `AC-*`: `AC-002`, `AC-003`, `AC-004`
- Связанные `DEC-*`: `DEC-001`, `DEC-002`
- Persisted data model не меняется.

## Сущности

### DM-001 PromptContextLimits (лимиты контекста)

- Назначение: задавать верхние границы для секции “Контекст:” в prompt `Answer*`.
- Источник истины: `PipelineOptions` (pkg) и `PipelineConfig` (internal).
- Инварианты:
  - `MaxContextChunks >= 0`, `0` = без лимита.
  - `MaxContextChars >= 0`, `0` = без лимита.
  - если оба лимита включены — соблюдаются оба.
- Связанные `AC-*`: `AC-002`, `AC-003`, `AC-004`
- Связанные `DEC-*`: `DEC-001`, `DEC-002`
- Поля:
  - `MaxContextChunks` — `int`, optional.
  - `MaxContextChars` — `int`, optional.

### DM-002 PromptContractV1 (секция контекста)

- Назначение: детерминированно формировать user message для LLM.
- Источник истины: application builder.
- Инварианты:
  - формат заголовков неизменен: “Контекст:” и “Вопрос:”.
  - лимиты применяются только к контексту между заголовками.
- Связанные `AC-*`: `AC-002`, `AC-003`, `AC-004`
- Поля:
  - `contextText` — `string`, derived (обрезанный по лимитам).
  - `question` — `string`, required.

## Связи

- `DM-001 -> DM-002`: лимиты определяют, сколько контента попадёт в `contextText`.

## Производные правила

- Сначала применяем `MaxContextChunks`, затем `MaxContextChars` (включая обрезание внутри последнего чанка).
- При выключенных лимитах поведение идентично текущему.

## Переходы состояний

- Не применимо: pipeline stateless.

## Вне scope

- Токен-ориентированные лимиты и адаптация под модели.
