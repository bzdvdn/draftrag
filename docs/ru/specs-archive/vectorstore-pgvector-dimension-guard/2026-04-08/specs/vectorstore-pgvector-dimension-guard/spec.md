# VectorStore pgvector: guardrails для размерности эмбеддингов (v1)

## Scope Snapshot

- In scope: добавить проверку размерности векторов (embedding dimension) на уровне pgvector-реализации, чтобы ошибки конфигурации ловились рано и были диагностируемы.
- Out of scope: автоматическое определение размерности модели по сети, поддержка нескольких размерностей в одной таблице, миграции данных при смене размерности.

## Цель

Предотвратить скрытые ошибки retrieval при несоответствии размерности эмбеддингов между Embedder и pgvector схемой/конфигом.

## Основной сценарий

1. Пользователь конфигурирует Embedder (например 1536-dim) и pgvector store.
2. При `Upsert`/`Search` библиотека проверяет размерность вектора.
3. При несоответствии возвращается понятная ошибка (а не “SQL error” или тихое ухудшение качества).

## Контекст

- В `PGVectorOptions` уже присутствует `EmbeddingDimension`, а `PGVectorStore` хранит `embeddingDim` и валидирует embedding в `Upsert` и `Search`.
- Сейчас ошибки несоответствия размерности могут выглядеть как “обычная” ошибка валидации/SQL, без явного классификатора, который удобно проверять через `errors.Is`.
- Требуется явная, типизированная ошибка “dimension mismatch”, чтобы пользователи могли отличать проблемы конфигурации от сетевых/SQL/логических ошибок.

## Требования

- RQ-001 В конфигурации pgvector store ДОЛЖНА быть явная `Dimension` (int) для v1.
- RQ-002 При `Upsert` ДОЛЖНА быть проверка: `len(embedding) == Dimension`, иначе ошибка валидации.
- RQ-003 При `Search` ДОЛЖНА быть проверка: `len(queryEmbedding) == Dimension`, иначе ошибка валидации.
- RQ-004 Ошибка ДОЛЖНА быть типизированной/сравнимой через `errors.Is` (чтобы пользователь мог различать class of errors).
- RQ-005 Тесты ДОЛЖНЫ покрывать оба случая (Upsert и Search) без внешней сети.

## Критерии приемки

### AC-001 Ошибка на несоответствие размерности

- **Given** pgvector store сконфигурирован с `Dimension = N`
- **When** вызывается `Upsert` или `Search` с embedding длины `!= N`
- **Then** метод возвращает ошибку, сравнимую через `errors.Is`

### AC-002 Нормальный путь не ломается

- **Given** embedding длины `== Dimension`
- **When** вызывается `Upsert`/`Search`
- **Then** поведение соответствует текущей реализации (без дополнительных побочных эффектов)
